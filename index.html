CSV XML JSON
Struktura:
CSV to format tekstu, w którym dane są przechowywane w formie tabelarycznej, a każda linijka reprezentuje rekord,
a kolumny są oddzielone przecinkami.
Prostota:
Jest prosty i łatwy do zrozumienia, co sprawia, że jest często używany do przechowywania danych tabelarycznych.
Brak Struktury:
W przeciwieństwie do XML i JSON, CSV nie posiada wbudowanej struktury ani informacji o typach danych.
Jest bardziej płaski i nie zawiera hierarchii.
Zastosowania:
CSV jest powszechnie używany w arkuszach kalkulacyjnych, bazach danych, a także jako format eksportu/importu danych.
 		Struktura:
XML to język znaczników, co oznacza, że dane są przechowywane w postaci tekstu z wykorzystaniem znaczników
otwierających i zamykających.
Jest to samodzielny format, co oznacza, że każdy dokument XML zawiera informacje o swojej strukturze,
dzięki czemu jest bardziej czytelny dla ludzi.
Rozszerzalność:
XML jest "rozszerzalny", co oznacza, że możesz definiować niestandardowe znaczniki i struktury danych,
co sprawia, że jest bardziej elastyczny.
Zastosowania:
XML jest szeroko stosowany w różnych dziedzinach, takich jak
przetwarzanie dokumentów, konfiguracja aplikacji, przesyłanie danych między aplikacjami, itp.		 Struktura:
JSON to lekki format danych, zapisywany jako obiekt JavaScript.
Składa się z par klucz-wartość i może reprezentować różne typy danych, takie jak
liczby, ciągi znaków, tablice, obiekty, itp.
Czytelność:
Jest łatwy do zrozumienia dla ludzi, a jednocześnie łatwy do przetwarzania przez maszyny.
Rozszerzalność:
Podobnie jak XML, JSON jest elastyczny i umożliwia definiowanie niestandardowych struktur danych.
Zastosowania:
Jest szeroko używany w przeglądarkach internetowych do przesyłania danych między klientem a serwerem (AJAX),
w aplikacjach internetowych, a także jako format wymiany danych w API.
